# S6.3 MINI-SMOKE (macOS) — kill-switch fail-closed + AUTO-recovery

Scope: prove the pf kill-switch on a real Mac — (1) it BLOCKS cleartext while the
helper is dead, and (2) the host **auto-recovers** (never stranded). ~20 min.

> ⚠️ **Designed to be UNSTRANDABLE.** Requires a gateway that EGRESSES (below), so
> full-tunnel UP = working internet — never a black hole. The kill uses the INSTALLED
> KeepAlive daemon: after `kill -9`, launchd restarts the helper within ~10 s and its
> startup self-heal releases the block **automatically**. You should not need to touch
> anything to recover. Recovery paths are unit-tested (`TestSupervisorSelfHeal`,
> `TestSupervisorDeadMan`).

## 🔴 If ever stuck: `sudo pfctl -a tunnex -F all` flushes the block. If the tunnel is UP, first bring it DOWN (Ctrl-C `tunnelctl`) — the *routes*, not the block, carry your traffic while it's up.

---

## Prereq A — gateway MUST egress (one-time, on the VM)
Full-tunnel routes ALL traffic to the gateway; without NAT it black-holes. Enable it:
```sh
# on the VM (needs sysctls: net.ipv4.ip_forward=1 in docker-compose.yml for node-agent)
docker compose exec node-agent cat /proc/sys/net/ipv4/ip_forward     # must be 1
docker compose exec node-agent sh -c '
  apk add --no-cache iptables >/dev/null 2>&1
  EG=$(ip route show default | awk "{print \$5; exit}")
  iptables -t nat -C POSTROUTING -s 10.99.0.0/24 -o "$EG" -j MASQUERADE 2>/dev/null || iptables -t nat -A POSTROUTING -s 10.99.0.0/24 -o "$EG" -j MASQUERADE
  iptables -C FORWARD -i wg0 -o "$EG" -j ACCEPT 2>/dev/null || iptables -A FORWARD -i wg0 -o "$EG" -j ACCEPT
  iptables -C FORWARD -i "$EG" -o wg0 -m state --state RELATED,ESTABLISHED -j ACCEPT 2>/dev/null || iptables -A FORWARD -i "$EG" -o wg0 -m state --state RELATED,ESTABLISHED -j ACCEPT
  echo NAT-ok'
```

## Prereq B — on the Mac
```sh
cd ~/tunnex && git pull       # story/S6.3-tunnel-control (endpoint-route + recovery fixes)
```

## 0. Full-tunnel config (Tab A)
```sh
( cd apps/cli && GOFLAGS=-mod=readonly go build -o /tmp/tunnex ./cmd/tunnex )
/tmp/tunnex login --server http://<server>
/tmp/tunnex device create --name mini-smoke --full-tunnel
CONF=~/.config/tunnex/device.conf
grep AllowedIPs "$CONF"        # must be: AllowedIPs = 0.0.0.0/0, ::/0  (never hand-edit)
```

## 1. Install helper — INSTALLED daemon (KeepAlive), short dead-man
```sh
./scripts/macos-dev-install.sh          # STEP ZERO must print caller-auth: native
# give the daemon a short auto-release window for the test:
sudo launchctl setenv TUNNEX_DEADMAN 15s 2>/dev/null || true
sudo launchctl kickstart -k system/io.tunnex.helper
```
Do NOT bootout the daemon this time — its auto-restart is the recovery.
Note your uplink BEFORE tunneling: `route -n get 1.1.1.1 | awk '/interface/{print $2}'` → usually `en0`.

## 2. Tunnel up — verify internet WORKS through it (Tab A holds)
```sh
sudo /usr/local/tunnex/tunnelctl up "$CONF" --full     # {"ok":true,"state":"up"}, holds
```
Tab B:
```sh
ping -c3 1.1.1.1        # REPLIES = full-tunnel carries real internet (endpoint route + gateway NAT)
```
**If this replies, `up` = working internet — no strand.** (`tx` should be small + steady, NOT
exploding — an exploding `tx` with `rx≈0` means the endpoint route is missing; Ctrl-C and report.)
Clean down anytime: **Ctrl-C `tunnelctl`**.

## 3. KILL-SWITCH — kill the daemon, prove ZERO leak, watch it AUTO-recover (Tab B)
```sh
UPLINK=en0
T=1.1.1.1; T6=2606:4700:4700::1111
ping -i 0.2 $T  >/dev/null &  P4=$!
ping -i 0.2 $T6 >/dev/null &  P6=$!
sudo tcpdump -i "$UPLINK" -n host $T  -w /tmp/ks4.pcap &  D4=$!
sudo tcpdump -i "$UPLINK" -n host $T6 -w /tmp/ks6.pcap &  D6=$!
sleep 2
HPID=$(sudo launchctl print system/io.tunnex.helper | awk '/pid =/{print $3}')
date; sudo kill -9 "$HPID"          # utun dies (routes revert); pf block persists (kernel-resident)
# launchd restarts the helper within ~10s → CleanStale self-heals → internet returns on its own.
echo "waiting for auto-recovery…"; for i in $(seq 1 20); do ping -c1 -t1 $T >/dev/null 2>&1 && { echo "recovered after ~${i}s"; break; }; sleep 1; done
sudo kill $D4 $D6 $P4 $P6 2>/dev/null
echo "--- v4 pkts DURING the dead window (should be 0) ---"; sudo tcpdump -r /tmp/ks4.pcap -n 'host 1.1.1.1 and not udp' | head
echo "--- v6 pkts DURING the dead window (should be 0) ---"; sudo tcpdump -r /tmp/ks6.pcap -n 'host 2606:4700:4700::1111 and not udp' | head
```
**PASS (all):**
- `recovered after ~Ns` printed (auto-recovery via KeepAlive restart + self-heal) — **you never touched anything**.
- Both pcaps show ZERO cleartext to the targets in the window between the kill and recovery
  (the helper was GONE, yet nothing leaked). One packet = FAIL.

## 4. Cleanup
```sh
sudo pkill -f tunnelctl 2>/dev/null; sudo pfctl -a tunnex -F all 2>/dev/null || true
ping -c2 1.1.1.1        # normal internet
```
