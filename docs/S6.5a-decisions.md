## A. The signed-vs-unsigned residue-smoke question — RESOLVED (definitive, per platform)

Carried from the S6.3 merge: *can the packaged install→run→uninstall residue smoke run
against the UNSIGNED S6.5a package, given SMAppService (macOS) / SCM registration
(Windows) may require a signed daemon?* Answer is platform-split:

- **macOS via `SMAppService` → NO (defers to S6.5b).** `SMAppService.register()` validates
  the app bundle's **Developer ID** code signature; an unsigned / ad-hoc-signed bundle
  fails registration. So the SMAppService install/uninstall path — and its residue smoke —
  **cannot be exercised unsigned**. Trigger to run it = S6.5b (Developer ID signing). This
  matches what S6.3 already ledgered (`scripts/macos-dev-install.sh` comment: "the
  production SMAppService installer ships a real Developer ID signature — S6.5b").

- **macOS via the INTERIM manual-LaunchDaemon path → YES (runs at S6.5a).** The S6.3 dev
  lifecycle installs an **ad-hoc-codesigned** helper at
  `/Library/LaunchDaemons/io.tunnex.helper.plist` and loads it with `launchctl bootstrap
  system`. `launchctl` loads an ad-hoc-signed daemon fine — only **SMAppService** demands
  Developer ID. So S6.5a ships this interim lifecycle (behind an in-app GUI privileged
  install; see §D), and its residue smoke runs unsigned. **S6.5b swaps this for
  SMAppService** — same helper, different registration front-end.

- **Windows via SCM service → YES (runs at S6.5a, machine permitting).** Windows SCM does
  **not** enforce Authenticode on **user-mode** service binaries (only kernel-mode drivers
  need KMCS). So `CreateService` → run → `DeleteService` + the residue smoke run **unsigned**.
  SmartScreen only gates the installer's first-launch UX (§C), not service registration or
  execution.

Net: the residue smoke **is** exercisable at S6.5a on both platforms — on macOS against the
interim path (not SMAppService), on Windows against the real SCM service. Only the
**SMAppService-specific** macOS residue smoke defers to S6.5b.

## C. Gatekeeper / SmartScreen — a NAMED deliverable (goes in `docs/install.md`, not a footnote)

Unsigned means the OS actively blocks first launch. "Packaged" ≠ "installable by the people
you hand it to" unless the bypass is documented. `install.md` must spell out, per platform:

- **macOS (`.dmg`, unsigned + un-notarized):** double-click → Gatekeeper "cannot verify
  developer / may be malware". Documented bypasses, in order of preference:
  1. **`curl`-install avoids quarantine entirely** — files fetched via `curl`/`wget` get **no**
     `com.apple.quarantine` xattr (only browser/Finder downloads do), so a `curl -LO <url>`
     of the `.dmg` opens without the Gatekeeper prompt. Preferred hand-off path.
  2. **Right-click (Control-click) → Open → Open** — the classic per-app override (macOS ≤ 14).
  3. **System Settings → Privacy & Security → "Tunnex was blocked" → Open Anyway** — required
     on **macOS 15 Sequoia**, which removed the Control-click bypass for un-notarized apps.
  4. Manual: `xattr -dr com.apple.quarantine /Applications/Tunnex.app` (documented as the
     escape hatch, not the primary path — it is a terminal step).
- **Windows (`.exe` NSIS installer, unsigned, no EV reputation):** SmartScreen "Windows
  protected your PC" blue dialog → **"More info" → "Run anyway"**. Note that SmartScreen warns
  on every unsigned/low-reputation installer until signing + install volume build reputation
  (that is exactly what S6.5b buys); document it as expected, not a defect.

`install.md` also covers: **verify the download** against `SHA256SUMS` (the integrity
substitute for a signature until S6.5b), and the **helper-install consent** (why the app asks
for an admin password on first connect — it installs the root privilege helper; §D).

## D. Packaging scope

- **electron-builder**, targets: **macOS `dmg`** + **Windows `nsis` `.exe`**. (Auto-update
  channels/`zip`/`latest.yml` stay OFF — that's S6.5b.)
- **macOS arch: `universal`** (electron universal2 + the helper built for arm64 **and** amd64
  then `lipo`-merged) so one `.dmg` runs on Apple Silicon and Intel. *Fallback if the universal
  helper merge proves fiddly:* arm64-only `.dmg` for S6.5a (Pawan's machine + most modern Macs)
  with x64 as a fast follow — decide at build time, record which shipped.
- **Bundled helper:** the Go privilege helper (built **CGO=1** for native caller-auth, per
  arch) ships inside the app resources. macOS: staged for the LaunchDaemon install. Windows:
  staged for the SCM service install + wintun.
- **In-app GUI privileged install (the crux for "zero terminal"):** end users must NOT run
  `macos-dev-install.sh`. S6.5a adds an in-app first-connect **"install helper"** step:
  - **macOS:** a GUI admin-authorization prompt (`osascript … "with administrator privileges"`,
    or a tiny privileged installer) that ad-hoc-codesigns + copies the helper and
    `launchctl bootstrap`s the LaunchDaemon. Zero terminal for the user (they type a password
    in a GUI dialog). The dev scripts stay for development only.
  - **Windows:** the NSIS installer runs elevated (UAC) and registers the SCM service + wintun
    at install time — no per-connect prompt.
- **`SHA256SUMS`** over the shipped artifacts, published alongside them.
- **`docs/install.md`** — the §C bypasses + hash-verify + helper-consent, per platform.

## E. Acceptance test (the real bar)

**Nothing-installed machine → install → connect → ping, with ZERO terminal commands.**
- macOS: mount `.dmg` → drag Tunnex to Applications → launch past Gatekeeper (§C) → sign in →
  Connect → the app's GUI admin prompt installs the helper → tunnel up → **ping through the
  tunnel succeeds**. No `sudo`, no script.
- Windows: run `.exe` → past SmartScreen (§C) → UAC-elevated install registers the service +
  wintun → launch → sign in → Connect → **ping through the tunnel succeeds**. No terminal.

The macOS in-app privileged install (§D) is the **primary risk** — it is the piece that makes
"zero terminal" real on an unsigned build. If it can't be made reliable unsigned, that is a
commit-one finding to surface, not a silent slip.
