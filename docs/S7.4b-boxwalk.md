# S7.4b — differentiated health surface, box-walk (X-3 wire-legs on the two-gateway env)

Proof env = the standing two-gateway enterprise box (G1 `demo-gw`, G2 `g2`), enforcing policy.
The badge reads `Node.policy_degraded_kind` (advisory over the authoritative `policy_degraded`
bool). `PSQL()`/`AUTH`/env = the S7.3/S7.4a STEP-0 blocks.

## Boot-log (rider a) — operator discoverability
```bash
docker compose logs api 2>&1 | grep policy_health_tuning
#  EXPECT: policy_health_tuning assumed_report_interval=30s desync_settle_window_T=60s report_freshness_window_F=60s
```

## Leg 1 — CONVERGING → HEALTHY (the naturally-stageable wire-leg)
A real policy change briefly diverges pushed≠applied until the agent applies + reports (≤ one
report cycle, 30s), so `policy_desync_since` is STAMPED then CLEARED — the badge flashes
"syncing…" (converging, amber, NOT a loud alarm) then clears (no badge / healthy).
```bash
# watch the CP-owned onset column for G1 while you add/delete a rule in the UI:
G1=019f5a14-1c1b-7343-bfb9-76e94a54b574
while true; do PSQL "SELECT policy_desync_since FROM nodes WHERE id='$G1'"; sleep 2; done
#  add or delete a rule (Access UI) → the column goes NON-NULL (stamped) then back to NULL
#  (cleared) on the next report ≤30s. In the Gateways UI the badge shows "syncing…" then clears.
```
**PASS:** onset stamped then cleared within ~a report cycle; converging badge is subtle, never red;
never `silent_desync` inside T.

## Leg 2 — SILENT_DESYNC (past-T, fresh reports) — SIMULATED (agent applies correctly by construction)
A working agent applies every push, so a *persistent fresh* mismatch can't arise naturally. Simulate
the state and confirm the render, then confirm it clears on real reconvergence:
```bash
# force a stale applied hash + an old onset on G1 (fresh last_seen), with enforcing pushed != applied:
PSQL "UPDATE nodes SET capabilities = jsonb_set(capabilities,'{policy_hash}','\"STALE\"'),
      policy_desync_since = now() - interval '90 seconds', last_seen_at = now() WHERE id='$G1'"
curl -s "${AUTH[@]}" "$BASE/api/v1/organizations/$ORG/nodes" | jq '.[]|select(.id=="'$G1'")|.policy_degraded_kind'
#  EXPECT: "silent_desync"  (badge: red "silent desync")
#  cleanup — let the agent's next real report re-stamp truthfully, or:
PSQL "UPDATE nodes SET policy_desync_since = NULL WHERE id='$G1'"
```
**PASS:** past-T + fresh + mismatch renders `silent_desync`. (The onset arithmetic T=60s is unit-pinned;
this leg confirms the render consumes the column on the wire.)

## Leg 3 — DESYNC_UNKNOWN (reports stop) — the headline honest state ★
`desync_unknown` = can't-determine. Two causes; the wire-leg is **stop a desynced gateway's reports**:
a stamped node that goes silent must read `desync_unknown` (NOT healthy, NOT silent_desync — silence
is not evidence). NOTE: a HEALTHY node that goes silent correctly stays *healthy* (its last report said
in-sync; silence is a liveness concern shown by `last seen`, not a policy-health one).
```bash
# stamp a desync on G2 + make its reports stale (simulate the gateway going silent WHILE desynced):
G2=019f5ef6-d583-7fad-b345-ccda36431b36
PSQL "UPDATE nodes SET capabilities = jsonb_set(capabilities,'{policy_hash}','\"STALE\"'),
      policy_desync_since = now() - interval '5 minutes',
      last_seen_at = now() - interval '5 minutes' WHERE id='$G2'"
curl -s "${AUTH[@]}" "$BASE/api/v1/organizations/$ORG/nodes" | jq '.[]|select(.id=="'$G2'")|.policy_degraded_kind'
#  EXPECT: "desync_unknown"  (badge: grey "health unknown") — NEVER "healthy", NEVER "silent_desync".
#  Optional real version: `docker stop tunnex-node-agent-g2` after forcing a mismatch, wait > F(60s).
PSQL "UPDATE nodes SET policy_desync_since = NULL WHERE id='$G2'"   # cleanup
```
**PASS:** stamped + stale reports → `desync_unknown`, never healthy/silent.
