# S7.3 box proof — copy-paste command sheet

Concrete commands for `docs/S7.3-boxproof-runbook.md`. Run on the gateway box (enterprise
stack up). Fill the vars in **STEP 0** once; every later block uses them.

**Enterprise stack (reproducible from git — no Dockerfile edit):** device approval is enterprise-gated,
so bring the stack up with `make up-enterprise` (== `TUNNEX_BUILD_TAGS=enterprise docker compose up -d
--build`). Verify `curl -s localhost/api/v1/meta | grep -o '"edition":"[a-z]*"'` prints `enterprise`.
The default `make up` builds the OPEN image (the edition-isolation CI guard runs on that default).

## STEP 0 — env (set once, in the shell you'll run curls from)

```bash
# API base — the tenant this box serves (adjust host/scheme to your deploy)
export BASE="https://app.tunnex.local"        # or http://localhost:8080 behind nginx

# Auth: a bearer for an OWNER/ADMIN of the org (device:approve needs owner|admin).
# Grab it the same way as the S7.2 run (browser devtools -> Authorization header,
# or your CLI token). Every mutating call also sends the CSRF header.
export TOKEN="paste-owner-bearer-here"
AUTH=(-H "Authorization: Bearer $TOKEN" -H "X-Tunnex-CSRF: 1" -H "Content-Type: application/json")

# Org id + the enrolled gateway node id (pulled from the API):
export ORG="$(curl -s "${AUTH[@]}" "$BASE/api/v1/organizations" | jq -r '.[0].id')"
export NODE="$(curl -s "${AUTH[@]}" "$BASE/api/v1/organizations/$ORG/nodes" | jq -r '.[0].id')"
echo "ORG=$ORG NODE=$NODE"

# nft/tcpdump inside the node-agent netns:
NFT() { sudo nsenter -t "$(docker inspect -f '{{.State.Pid}}' "$(docker compose ps -q node-agent)")" -n "$@"; }

# psql into the API's postgres (adjust service name if different):
PSQL() { docker compose exec -T postgres psql -U postgres -d tunnex -tAc "$1"; }
```

Sanity: `NFT wg show wg0` prints the gateway interface; `echo $ORG` is a uuid.

## LEG 1 — device_approval=ON: new device enrolls PENDING, cannot connect

```bash
# 1. Turn approval ON — note grandfathered_count in the response
curl -s "${AUTH[@]}" -X PUT "$BASE/api/v1/organizations/$ORG/device-approval" \
  -d '{"mode":"on"}' | jq

# 2. Enroll a device for spoke A (server-generates the key -> one-time .conf).
#    Saves the .conf to A.conf and captures pending_approval + the device id.
RESP="$(curl -s "${AUTH[@]}" -X POST "$BASE/api/v1/organizations/$ORG/devices" \
  -d "{\"name\":\"spokeA\",\"node_id\":\"$NODE\"}")"
echo "$RESP" | jq '{pending_approval, id: .device.id, status: .device.status}'
echo "$RESP" | jq -r '.config' > A.conf
export DEV_A="$(echo "$RESP" | jq -r '.device.id')"
#    EXPECT: pending_approval=true, status="pending"

# 3. Bring spoke A up with the issued config (netns spoke)
sudo ./deploy/testing/s72-spokes.sh up A.conf A.conf   # 2nd arg is a placeholder for B; re-run in Leg 3

# 4. PROVE it cannot connect (server boundary):
NFT wg show wg0                                          # EXPECT: A's pubkey is NOT a peer
sudo ./deploy/testing/s72-spokes.sh a ping -c3 10.99.0.1  # EXPECT: 100% loss
#    (optional, if enforcing) A's /32 in NO allow rule:
NFT nft list ruleset | grep "$(grep -i '^Address' A.conf | cut -d= -f2 | tr -d ' ' | cut -d/ -f1)" || echo "no rule (correct)"
```
**PASS (server):** no wg peer, ping 100% loss, no allow rule.

### Leg 1 (client half) — desktop connect-gate
5. Point the Tunnex desktop client at this tenant, sign in as spoke-A's user, click **Connect**.
6. EXPECT: UI shows **"Awaiting admin approval…"**; **no** admin/helper prompt; **no** `utun`
   (macOS `ifconfig | grep utun`) / no WFP adapter (Windows `Get-NetAdapter`); across **≥60s
   NO "revoked" notification**. (Absence of the spurious revoked = the fix.)

### Leg 1 (upgrader variant) — legacy re-mint + the new `migrate_failed` legible state ★
7. Quit the client. Hand-edit the persisted `StoredTunnelConfig` for this origin to **remove the
   `orgId` field** (simulates a pre-orgId v0.1.0 config). Config store path:
   - macOS: `~/Library/Application Support/Tunnex/tunnels.json`
   - Windows: `%APPDATA%\Tunnex\tunnels.json`
8. Reopen, click **Connect**. EXPECT: one-time **"Tunnex device replaced"** notification; a FRESH
   device is minted (old best-effort revoked); NOT cross-org queried. Under approval=on the fresh
   device is **pending** -> "Awaiting admin approval…".
9. **(new — the fifth-touch legibility proof)** To see the `migrate_failed` state: mute OS
   notifications, then force the migration revoke to fail (point the client at an unreachable
   server briefly, or block the revoke), Connect. EXPECT the **window/tray shows "Couldn't replace
   device — reconnect to retry"** (amber) — **not a bare "Disconnected"**; profile is kept;
   reconnect retries. *(If awkward on the box, the reds `migrateLegacyConfig: a revoke blip KEEPS
   the config` + `trayStateFor migrate_failed -> migrate_retry` cover it — note it, skip.)*

## LEG 2 — APPROVE -> connects < 5s (timed)

```bash
t0=$(date +%s.%N)
curl -s "${AUTH[@]}" -X POST "$BASE/api/v1/organizations/$ORG/devices/$DEV_A/approve" -o /dev/null -w "approve HTTP %{http_code}\n"
# poll for the peer + handshake
until NFT wg show wg0 | grep -q "$(grep -i PublicKey A.conf | tail -1)"; do :; done   # or eyeball NFT wg show wg0
sudo ./deploy/testing/s72-spokes.sh a ping -c3 10.99.0.1
t1=$(date +%s.%N); echo "Δ connect = $(echo "$t1 - $t0" | bc)s   (ASSERT < 5)"

# approved_by attributed + active:
curl -s "${AUTH[@]}" "$BASE/api/v1/organizations/$ORG/devices" | jq --arg d "$DEV_A" '.[]|select(.id==$d)|{status,approved_by}'
```
4b. Desktop (if watching the client on device A): approve fires the **"device approved — click
   Connect"** notification, device becomes **connectable**, tunnel does **NOT auto-connect**.

**PASS:** peer+ping < 5s; `approved_by` set; client notifies, no auto-connect.

## LEG 3 — REJECT -> teardown + IP freed for reuse

```bash
# Enroll a SECOND pending device (spoke B); note its assigned IP
RESP_B="$(curl -s "${AUTH[@]}" -X POST "$BASE/api/v1/organizations/$ORG/devices" -d "{\"name\":\"spokeB\",\"node_id\":\"$NODE\"}")"
export DEV_B="$(echo "$RESP_B" | jq -r '.device.id')"
export IP_B="$(echo "$RESP_B" | jq -r '.device.assigned_ip')"; echo "IP_B=$IP_B"

# Reject it -> revoked + IP null
curl -s "${AUTH[@]}" -X POST "$BASE/api/v1/organizations/$ORG/devices/$DEV_B/reject" -o /dev/null -w "reject HTTP %{http_code}\n"
curl -s "${AUTH[@]}" "$BASE/api/v1/organizations/$ORG/devices" | jq --arg d "$DEV_B" '.[]|select(.id==$d)|{status,assigned_ip}'
#    EXPECT: status="revoked", assigned_ip=null

# Enroll a THIRD device -> reuses IP_B (lowest-free)
RESP_C="$(curl -s "${AUTH[@]}" -X POST "$BASE/api/v1/organizations/$ORG/devices" -d "{\"name\":\"spokeB2\",\"node_id\":\"$NODE\"}")"
echo "$RESP_C" | jq --arg ip "$IP_B" '{reused: (.device.assigned_ip==$ip), assigned_ip: .device.assigned_ip}'
#    EXPECT: reused=true
```
**PASS:** reject -> revoked + IP freed; next enrollment reuses IP_B.

## LEG 3b — F1-part-3 org-wide push ON THE WIRE ★ (needs 2 gateways + enforcing)

Precondition: `zero_trust_mode=enforcing`; two gateways G1/G2; a policy rule whose DST group
contains the reject-target's owner, so its /32 is a group-dst grant on **G2** (which does NOT
host it). `NFT2()` = the same nsenter helper pointed at G2's node-agent.

```bash
# 1. Before: the grant exists on G2
NFT2 nft list ruleset | grep "$IP_B"                     # EXPECT: a group-dst rule line

# 2. TIMED: reject/revoke, poll G2 until the grant is gone
t0=$(date +%s.%N)
curl -s "${AUTH[@]}" -X POST "$BASE/api/v1/organizations/$ORG/devices/$DEV_B/reject" -o /dev/null -w "%{http_code}\n"
until ! NFT2 nft list ruleset | grep -q "$IP_B"; do :; done
t1=$(date +%s.%N); echo "Δ grant-gone on G2 = $(echo "$t1 - $t0" | bc)s   (ASSERT < 5)"

# 3-4. New device reuses IP_B, owned by a user NOT in that dst-group. From the SOURCE spoke on G2
#      that HELD the grant to the old dst-group, probe the reused IP BEFORE any new grant:
NFT2 nft list ruleset | grep -c default_drop            # note counter before
#   <run: source-side ping -c3 $IP_B from G2's granted spoke>   EXPECT: 100% loss
NFT2 nft list ruleset | grep default_drop               # EXPECT: counter ticked by the probe
NFT2 nft list ruleset | grep "$IP_B" || echo "no grant for new holder (correct)"
```
**PASS:** stale grant stripped on the non-hosting gateway Δ<5s; reused IP -> default_drop from the
source side; new holder inherited NO reachability.

## LEG 4 — flip ON grandfathers the fleet (no mass black-hole)

```bash
# Start with approval OFF and >=2 ACTIVE connected devices (e.g. approve A + one more first).
curl -s "${AUTH[@]}" -X PUT "$BASE/api/v1/organizations/$ORG/device-approval" -d '{"mode":"off"}' | jq -r .mode

# 2. CONTINUOUS ping on a grandfathered device (leave running in another shell):
sudo ./deploy/testing/s72-spokes.sh a ping 10.99.0.1     # count the loss across the flip

# 3. Flip ON — grandfathered_count must equal the active count
ACTIVE="$(PSQL "SELECT count(*) FROM devices WHERE org_id='$ORG' AND status='active';")"
curl -s "${AUTH[@]}" -X PUT "$BASE/api/v1/organizations/$ORG/device-approval" -d '{"mode":"on"}' \
  | jq --argjson a "$ACTIVE" '{grandfathered_count, active:$a, match:(.grandfathered_count==$a)}'

# 4. Stop the ping (Ctrl-C in its shell) -> EXPECT 0% packet loss across the flip.

# 5. grandfathered rows have approved_by NULL:
PSQL "SELECT count(*) FROM devices WHERE org_id='$ORG' AND status='active' AND approved_by IS NULL;"
#    EXPECT: == the grandfathered fleet count

# 6. Negative: a fresh enrollment AFTER the flip lands pending
curl -s "${AUTH[@]}" -X POST "$BASE/api/v1/organizations/$ORG/devices" -d "{\"name\":\"postflip\",\"node_id\":\"$NODE\"}" \
  | jq '{pending_approval, status:.device.status}'      # EXPECT: pending_approval=true
```
**PASS:** grandfathered_count==fleet; ZERO lost packets across flip; grandfathered rows
`approved_by IS NULL`; new enrollment gates pending.
