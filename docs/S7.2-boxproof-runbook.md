# S7.2 ‚Äî Zero Trust enforcement: on-the-wire box-proof runbook

**Status: DRAFT for sign-off ‚Äî do NOT run until approved.** This is the S7.2 DoD gate. The unit +
integration tests SUBSTITUTE for but DO NOT satisfy these; a real packet must be DROPPED on a real
gateway, same bar as the S6.3/S6.7 kill-switch pcaps.

## Topology

- **1 gateway** (`tunnex-node`, Linux, `nft` present) enrolled to an **enterprise** control plane.
- **2 spoke devices** A + B, both connected through the gateway, owner-bound to org members:
  - A `10.99.0.10`, owner Alice; B `10.99.0.20`, owner Bob.
- A **3rd host** C behind the gateway (a plain LAN box, e.g. `10.0.5.5`) for resource/egress grants.
- Terminal on the gateway + on A.

## Pre-run confirmations (before any proof)

- **`egress_nat=true` on the gateway** ‚Äî verify the node reports egress-capable (its capabilities /
  the Gateways list) BEFORE proof 4. Otherwise proof 4's "no internet" would prove NAT ABSENCE, not
  POLICY denial ‚Äî the whole egress-default-deny claim would be confounded.
- Both editions of the "surface": for proof 7, STALE is observed on **the node's stored capabilities**
  (`policy_hash`, `policy_error`, `policy_failing_since`) on the control plane ‚Äî queried via the nodes
  row / the Gateways status. `STALE` = `pushed_hash ‚âÝ applied policy_hash` persisting > 90s
  (`nodes.NodeCapabilities.PolicyStale`); a first-class dashboard STALE badge is S7.4, but the DATA +
  the predicate are S7.2 and observable here.

## Instrumentation

- **Ruleset view:** `nft list ruleset` (the forward chain of `table ip tunnex`). Every rule carries a
  `counter packets N bytes M` (S7.2 day-one counters) ‚Äî this is the load-bearing observation:
  - each `ip saddr ‚Ä¶ ip daddr ‚Ä¶ counter accept` = an ALLOW counter (positive proof).
  - the trailing `counter comment "tunnex_default_drop"` = the DEFAULT-DENY counter (negative proof):
    packets that matched no allow and are about to hit `policy drop`.
- **Wire capture:** `tcpdump -ni wg0 host <dst>` on the gateway (a drop = the packet arrives on wg0 in
  but is NOT forwarded out).
- **Applied-policy status:** the node's report carries `policy_version`, `policy_hash` (canonical),
  `policy_error`; the control plane exposes it (node capabilities). Used for proof 7.

## Timing methodology (the <5s legs: proofs 5, 6, 8)

- **t0 = the control-plane mutation commit.** Wrap the API call: `T0=$(date +%s.%N); curl ‚Ä¶ ; ` ‚Äî t0 is
  the instant the mutation's HTTP 2xx returns (the row is committed + the push fired).
- **t1 = the first on-the-wire observation of the new state**, whichever the proof names:
  - a previously-passing flow's FIRST DROP: `tcpdump` shows the packet stop being forwarded, OR the
    `tunnex_default_drop` counter starts incrementing for that flow while the allow counter stops.
  - a previously-dropped flow's FIRST PASS: `tcpdump` shows it forwarded + the allow counter ticks.
- **latency = t1 ‚àí t0**, asserted **< 5s**. Run A's `ping`/`hping3` at a fixed interval (e.g. 0.2s) so
  t1 has ‚â§0.2s granularity. Record both timestamps + the delta per proof.

## The 8 proofs

### 1. default-deny on the wire (THE load-bearing one)
Org `enforcing`, **zero grants**. From A: `ping 10.99.0.20` (B) and `ping 10.0.5.5` (C).
**PASS:** both time out; `tcpdump -ni wg0` shows the ICMP echoes ARRIVE at the gateway but are NOT
forwarded; the `tunnex_default_drop` **counter increments** by the echo count. Device-to-device +
egress are denied by default.
**Enable-ceremony artifacts (S7.1 toggle conditions, first live exercise ‚Äî capture for free):**
- the **`org.zero_trust_enabled` audit row**, actor-attributed (the admin who flipped it);
- the **`affected_full_tunnel_devices` enumeration** in the enable-mode PUT response (count + names) ‚Äî
  the honest blast radius the server returned while OBEYING the flip (2a). Save both.

### 2. grant works (positive/negative PAIR)
Dashboard: group **admins**={Alice}, **servers**={Bob}, rule `admins ‚Üí servers`. Within the push
window, from A: `ping 10.99.0.20`.
**PASS:** replies; `nft list ruleset` shows `ip saddr 10.99.0.10 ip daddr 10.99.0.20/32 counter accept`
with its **allow counter incrementing**; `tcpdump` shows the echo forwarded. Simultaneously A‚ÜíC
(`ping 10.0.5.5`, no grant) still DROPS and the `tunnex_default_drop` counter ticks ‚Äî the granted flow
passes AND the non-granted flow drops, same capture window.

### 3. no blanket under enforcing (S7.1 structural guard, on the box)
While `enforcing`, `nft list ruleset`.
**PASS:** the forward chain has **NO `iifname "wg0" oifname "wg0" ‚Ä¶ accept`** (the blanket) ‚Äî only
`policy drop`, `ct established,related accept`, the explicit `ip saddr ‚Ä¶ counter accept` allows, and
the trailing `tunnex_default_drop` counter.

### 4. resource + egress grant (egress default-deny red)
Resource `db`=`10.0.5.0/24 tcp:5432`, rule `admins ‚Üí db`: from A `nc -z 10.0.5.5 5432`. Then resource
`internet`=`0.0.0.0/0`, rule `admins ‚Üí internet`: from A `curl -sS https://1.1.1.1`.
**PASS:** the DB port reachable ONLY after its grant; internet works ONLY after the `0.0.0.0/0` grant
(masquerade NATs it). **2-red (egress default-deny):** BEFORE the `0.0.0.0/0` grant, full-tunnel A has
NO internet ‚Äî `curl` times out, `tunnex_default_drop` ticks for the egress-bound packets.

### 5. revocation latency <5s ‚Äî device revoke
A‚ÜîB granted + flowing (A pings B at 0.2s). **t0** = revoke B's device in the dashboard (HTTP 2xx).
**t1** = A‚ÜíB's first dropped ping (`tcpdump` / counter).
**PASS:** `t1 ‚àí t0 < 5s`; afterwards **`nft list ruleset | grep 10.99.0.20` ‚Üí EMPTY** ‚Äî B's `/32` is
gone as **saddr AND daddr** (not merely unreachable ‚Äî physically absent from every allow-set). Record
t0, t1, Œî + the empty grep.

### 6. revocation latency <5s ‚Äî ORG-MEMBER removal (F1 4th trigger)
**Setup (load-bearing for F1):** Bob is a member of a group used as a rule **DESTINATION** before
removal ‚Äî so his `/32` is compiled into an allow-set as **dst** (e.g. `admins ‚Üí bob's-group`, A in
admins). Restore reach (A pinging B at 0.2s). **t0** = **remove Bob from the org** (not the device;
HTTP 2xx). **t1** = A‚ÜíB's first dropped ping.
**PASS:** `t1 ‚àí t0 < 5s` AND **`nft list ruleset | grep 10.99.0.20` ‚Üí EMPTY** ‚Äî Bob's `/32` leaves the
compiled ruleset on the wire as **saddr AND daddr** though his device was never individually revoked
(cascade-FK + PushOrgNodes path). This closes the F1 class ON THE WIRE, not just by reachability.
Record t0, t1, Œî + the empty grep.

### 7. staleness surfacing + the LIVE kernel-rollback proof (deliberate break)
Setup: A‚ÜîB granted + flowing under a known-good ruleset; note the current `policy_hash` (call it H0)
and `nft list ruleset` (R0). **Break the next apply**: on the gateway, `chmod -x $(command -v nft)`
(the agent's `nft -f` will now fail). Then push a policy change (e.g. add a rule) so the control plane
sends a NEW compiled hash H1.
- **EXPECTED FAILURE OBSERVATION (the apply failed AND the previous ruleset stayed ‚Äî the atomic-swap
  kernel-rollback claim, proven live):**
  1. the agent's next status report shows **`policy_error` non-empty** (the `nft -f` error) ‚Üí
     surfaced IMMEDIATELY;
  2. the reported **`policy_hash` is still H0** (applied did NOT advance to H1 ‚Äî fail-closed);
  3. on the box, `nft list ruleset` is **still R0** (the kernel rejected the failed batch wholesale ‚Äî
     the previous ruleset stayed in force, NO empty-chain window);
  4. A‚ÜîB **still flows** (the good ruleset is intact ‚Äî no leak, no lockout).
- **STALE alarm ‚Äî timed BOTH ways** (surface: the node's stored capabilities ‚Äî `policy_hash` /
  `policy_failing_since` on the control plane, `PolicyStale` predicate). **t0** = the failed-push HTTP 2xx
  (H1 pushed). Poll `PolicyStale`:
  - **t_stale** = when it first flips true. **Assert `t_stale ‚àí t0 ‚âà 90s` AND NOT before** ‚Äî sample at
    e.g. t0+60s (must be FALSE ‚Äî the transient window is not an alarm) and t0+95s (must be TRUE). A
    premature alarm inside 90s is a FAILURE. Record t0, t_stale.
- **RECOVERY:** `chmod +x $(command -v nft)` ‚Üí within the reconcile interval the agent applies H1,
  reports `policy_hash = H1`, `policy_error` clears, and `PolicyStale` clears.

### 7b. flush ‚Üí reconcile restores (S3.7-pattern red, desired-state never assumed)
While `enforcing` with A‚ÜîB flowing: on the gateway, `nft flush chain ip tunnex forward`.
- **EXPECTED FAILURE OBSERVATION (the flush took effect):** `nft list ruleset` shows the forward chain
  now holds ONLY `policy drop` (base) ‚Äî **the `ip saddr ‚Ä¶ counter accept` allow rules + the ct-accept
  are GONE**; A‚ÜíB **DROPS** (no allow counter; the packet hits policy drop).
- **RECOVERY:** within the reconcile interval the agent re-applies ‚Üí the allow rules + ct-accept
  REAPPEAR in `nft list ruleset` and A‚ÜîB flows again. Reconcile restored desired state; it was never
  assumed in sync.

### 8. mixed-mode transition (timed both directions)
`off ‚Üí enforcing` (with the admins‚Üíservers grant present): A‚ÜîB flowing under the legacy mesh; A‚ÜíC also
flowing (mesh blanket). **t0** = flip to `enforcing`. **t1(keep)** = A‚ÜîB still flowing (granted ‚Üí
uninterrupted); **t1(drop)** = A‚ÜíC's first drop.
**PASS:** A‚ÜîB never breaks (granted); A‚ÜíC starts dropping with `t1(drop) ‚àí t0 < 5s`; NO empty-chain
window ‚Äî A‚ÜîB has ZERO lost packets across the flip (the atomic-swap proof at the transition). Then
`enforcing ‚Üí off`: **t0'** = flip back; **t1'** = A‚ÜíC's first pass. **PASS:** blanket mesh restored,
`t1' ‚àí t0' < 5s` (fast lockout-recovery).
