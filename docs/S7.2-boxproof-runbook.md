# S7.2 — Zero Trust enforcement: on-the-wire box-proof runbook

**Status: DRAFT for sign-off — do NOT run until approved.** This is the S7.2 DoD gate. The unit +
integration tests SUBSTITUTE for but DO NOT satisfy these; a real packet must be DROPPED on a real
gateway, same bar as the S6.3/S6.7 kill-switch pcaps.

## Topology

- **1 gateway** (`tunnex-node`, Linux, `nft` present) enrolled to an **enterprise** control plane.
- **2 spoke devices** A + B, both connected through the gateway, both owner-bound to org members:
  - A `10.99.0.10`, owner Alice; B `10.99.0.20`, owner Bob.
- A **3rd host** C behind the gateway (a plain LAN box, e.g. `10.0.5.5`) to test resource/egress grants.
- Terminal access to the gateway (`nft list ruleset`, `pktmon`/`tcpdump` on `wg0`), and to A (ping/curl).

## Instrumentation

- Gateway: `watch -n1 'nft list ruleset | sed -n "/table ip tunnex/,/^}/p"'` and a capture
  `tcpdump -ni wg0 host 10.99.0.20 or host 10.0.5.5` (drops show as no forwarded packet + the forward
  `policy drop` counter incrementing).
- Timing: `date +%s.%N` bracketing each change → assert < 5s where the spec applies.

## The 8 proofs

### 1. default-deny on the wire (THE load-bearing one)
Org `enforcing`, **zero grants**. From A, `ping 10.99.0.20` (device B) and `ping 10.0.5.5` (host C).
**PASS:** both time out; `tcpdump -ni wg0` shows the ICMP echo arrive at the gateway but **no forward
to B/C**; `nft` forward drop counter increments. Device-to-device is denied by default.

### 2. grant works (positive control — pairs with 1, 3a)
Dashboard: group **admins**={Alice}, group **servers**={Bob}, rule `admins → servers`. Within the
push window, from A `ping 10.99.0.20`.
**PASS:** replies; `nft list ruleset` shows `ip saddr 10.99.0.10 ip daddr 10.99.0.20/32 accept`;
`tcpdump` shows the echo forwarded. (3a: allowed flow passes AND the non-granted flow to C still drops.)

### 3. no blanket under enforcing (3c — S7.1 structural guard, on the box)
While `enforcing`, `nft list ruleset`.
**PASS:** the forward chain contains **NO `iifname "wg0" oifname "wg0" accept`** (the blanket) — only
`policy drop`, `ct established,related accept`, and the explicit `ip saddr … ip daddr …` allows.

### 4. resource + egress grant
Resource `db`=`10.0.5.0/24 tcp:5432`, rule `admins → db`. From A, `nc -z 10.0.5.5 5432` (or curl).
Then a resource `internet`=`0.0.0.0/0`, rule `admins → internet`. From A, `curl -sS https://1.1.1.1`.
**PASS:** the DB port is reachable ONLY after its grant; internet works ONLY after the `0.0.0.0/0`
grant (the masquerade NATs it). **2-red:** BEFORE the `0.0.0.0/0` grant, a full-tunnel A has **no
internet** — `curl` times out, the drop counter ticks. Egress is default-deny under enforcing.

### 5. revocation latency <5s — device revoke (bracket with `date`)
With A↔B granted + flowing, revoke B's device in the dashboard. Time until A loses B.
**PASS:** A→B stops within **< 5s**; `nft` no longer lists B's `/32` (as src or dst).

### 6. revocation latency <5s — ORG-MEMBER removal (F1 4th trigger)
Re-add B; restore reach. Now **remove Bob from the org** (not just revoke the device). Time it.
**PASS:** A→B stops within **< 5s** — Bob's `/32` leaves the compiled ruleset on the wire even though
his device was never individually revoked (the cascade-FK + PushOrgNodes path). This is the F1 proof.

### 7. staleness surfacing (deliberate break)
On the gateway, make the next apply FAIL (e.g. `chmod -x $(command -v nft)` briefly, or inject a bad
rule via a temporary env), then change policy so a new hash is pushed.
**PASS:** the gateway keeps the PREVIOUS ruleset in force (kernel rollback — no leak), reports the
apply error, and the control plane surfaces the node as **STALE** once `pushed ≠ applied` persists
past **90s** (`policy_error` shows immediately). Restore `nft` → the node reconciles to fresh within
the interval. **3d red:** manually `nft flush chain ip tunnex forward` → the reconcile RESTORES the
compiled chain within the interval (desired-state, never assumed).

### 8. mixed-mode transition (timed both directions)
`off → enforcing`: with A↔B flowing under the legacy mesh, flip to `enforcing` with the admins→servers
grant ALSO present. **PASS:** the flip does not break the granted A↔B flow, but a NON-granted A→C flow
starts dropping — timed < 5s, no leak window (atomic swap). `enforcing → off`: flip back. **PASS:**
the blanket mesh is restored fast (< 5s), A→C reachable again — the fast lockout-recovery path.
