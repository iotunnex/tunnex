# S7.2 box proof — exact run guide (what / where / when / branch)

Companion to `docs/S7.2-boxproof-runbook.md` (the proof plan). This is the copy-paste procedure.

## Branch
Everything runs from **`story/S7.2-enforcement`** (PR #16 — has all the enforcement code). NOT main.

## Hosts (WHERE the commands run)
- **[GW]** = one Linux VM (Ubuntu 22.04+), the GATEWAY. Runs the whole stack + is where you observe
  `nft`/`tcpdump`. Needs: docker + compose plugin, `nftables`, `tcpdump`, a **public IP** (so the
  spokes can reach `:51820`).
- **[A]** = spoke device A — your existing desktop client (mac), device owned by "Alice".
- **[B]** = spoke device B — a second client (windows / second config), device owned by "Bob".
- **[CP]** = any host with `curl` to hit the API (can be [GW] itself: `localhost`).
- A/B use **split-tunnel** configs (AllowedIPs = pool CIDR) so A↔B routes THROUGH the gateway (that is
  what the forward chain gates). You have done device-to-device live before — same setup.

Tunnel addresses used below: **A = 10.99.0.10**, **B = 10.99.0.20**. (Adjust to whatever the pool
allocator actually assigns — read them off each device after it connects.)

## PHASE 0 — [GW] get the VM + branch (WHEN: first)
```bash
sudo apt-get update && sudo apt-get install -y nftables tcpdump git   # docker+compose already installed
git clone https://github.com/iotunnex/tunnex.git && cd tunnex
git checkout story/S7.2-enforcement
```

## PHASE 1 — [GW] bring up the ENTERPRISE stack (WHEN: after PHASE 0)
Zero Trust compiles ONLY under `-tags enterprise`, so the api image must be the enterprise build:
```bash
sed -i 's/go build /go build -tags enterprise /' deploy/docker/api.Dockerfile   # temporary; reverted in cleanup
cp .env.example .env
# .env: set APP_BASE_URL + TUNNEX_NODE_ENDPOINT to the VM's PUBLIC IP so spokes can dial in:
sed -i "s#APP_BASE_URL=.*#APP_BASE_URL=http://$(curl -s ifconfig.me)#" .env
sed -i "s#TUNNEX_NODE_ENDPOINT=.*#TUNNEX_NODE_ENDPOINT=$(curl -s ifconfig.me):51820#" .env
docker compose up -d --build
curl -s localhost/api/v1/meta | grep -o '"edition":"[a-z]*"'     # MUST print "enterprise"
```

## PHASE 2 — [GW] seed + enroll the gateway + confirm egress (WHEN: after stack is healthy)
```bash
make seed     # creates demo org + owner@demo.tunnex.local / tunnex-demo-password
```
- Browser → `http://<VM-public-ip>/` → sign in as `owner@demo.tunnex.local` / `tunnex-demo-password`.
- **Gateways → Generate join token → copy the ONE command → run it on [GW]** (it enrolls the local
  node-agent as the gateway).
- **PRE-RUN GATE — egress_nat must be true (before proof 4):**
```bash
docker compose exec postgres psql -U tunnex -d tunnex -c \
  "select name, capabilities->>'egress_nat' egress from nodes where status='active';"
#   egress = t   (if f: the VM has no default route / NAT — fix before proof 4)
```

## PHASE 3 — [A]/[B] create + connect the two spokes (WHEN: gateway enrolled)
- Dashboard → **Devices → add device A** (owner Alice) → download `A.conf` → import on [A], connect
  (split tunnel). Confirm on [A]: `ping 10.99.0.20` will work AFTER B is up (this is the mesh/off
  baseline).
- Dashboard → **Devices → add device B** (owner Bob) → `B.conf` → import on [B], connect.
- Baseline (mode still `off`): from [A] `ping 10.99.0.20` → **replies** (legacy mesh). Good — the
  before-state is proven working; enforcing must then DENY it.

## PHASE 4 — [GW] open the two observation terminals (WHEN: keep open for all proofs)
```bash
# terminal 1 — live forward chain + counters (this is your main instrument):
sudo watch -n1 'nft list ruleset | sed -n "/table ip tunnex/,/^}/p"'
# terminal 2 — wire capture on the WG interface INSIDE the node container:
CID=$(docker compose ps -q node-agent)
sudo nsenter -t $(docker inspect -f '{{.State.Pid}}' $CID) -n tcpdump -ni wg0 host 10.99.0.20 or host 10.0.5.5
```
> `nft`/`tcpdump` must run in the NODE CONTAINER's netns (that's where `wg0` + the tunnex table live).
> `nsenter -t <pid> -n <cmd>` enters it. Prefix the `nft`/`tcpdump`/`grep` commands below with that
> `sudo nsenter -t $(docker inspect -f '{{.State.Pid}}' $CID) -n` (export it once: `NS="sudo nsenter -t $(docker inspect -f '{{.State.Pid}}' $CID) -n"`).

## PHASE 6 — THE 8 PROOFS (run in order; record artifacts as you go)

**Enable enforcing + capture ceremony (proof 1 setup):** [CP]
```bash
curl -sb /tmp/cj -X PUT -H 'content-type: application/json' -H 'X-Tunnex-CSRF: 1' \
  -d '{"mode":"enforcing"}' localhost/api/v1/organizations/$ORG/zero-trust-mode
#   -> response JSON includes affected_full_tunnel_devices  [SAVE — proof 1 artifact]
docker compose exec postgres psql -U tunnex -d tunnex -c \
  "select action,actor_user_id,created_at from audit_logs where action='org.zero_trust_enabled' order by created_at desc limit 1;"   # [SAVE — proof 1 artifact]
```

**Proof 1 — default-deny:** [A] `ping 10.99.0.20`; `ping 10.0.5.5` → both TIME OUT. On [GW] watch the
`tunnex_default_drop` counter climb; `tcpdump` shows echoes arrive, not forwarded.

**Proof 2 — grant works (pos/neg pair):** Dashboard → Groups `admins`={Alice} `servers`={Bob}; Policy
rule `admins → servers`. [A] `ping 10.99.0.20` → REPLIES; the `ip saddr 10.99.0.10 ip daddr
10.99.0.20/32 counter accept` counter climbs. Same time [A] `ping 10.0.5.5` → still DROPS
(default_drop climbs).

**Proof 3 — no blanket:** [GW] `$NS nft list ruleset | grep 'oifname "wg0"'` → shows only the explicit
allows + masquerade, **NO `iifname "wg0" oifname "wg0" ... accept` blanket**.

**Proof 4 — resource + egress (egress-deny red):** Dashboard → Resource `db`=`10.0.5.0/24 tcp 5432` +
rule `admins→db`; [A] `nc -zv 10.0.5.5 5432` works. Then Resource `internet`=`0.0.0.0/0` + rule; [A]
`curl -m5 https://1.1.1.1` works. **Before** the internet grant (delete the rule): [A] `curl -m5
https://1.1.1.1` TIMES OUT + default_drop climbs.

**Proof 5 — device revoke <5s:** [A] run `ping -i 0.2 10.99.0.20` in a loop.
```bash
# [CP]:
T0=$(date +%s.%N); curl -sb /tmp/cj -X POST -H 'X-Tunnex-CSRF: 1' \
  localhost/api/v1/organizations/$ORG/devices/<B-device-id>/revoke; echo "t0=$T0"
# note wall-clock when A's pings start failing = t1; delta = t1 - t0  (< 5s)
# [GW]:
$NS nft list ruleset | grep 10.99.0.20     # -> EMPTY (gone as saddr AND daddr)
```

**Proof 6 — member removal <5s (F1):** re-add B; **make Bob's group a rule DST first** (e.g. rule
`admins → servers` already does — Bob in servers = a dst). [A] ping loop again.
```bash
# [CP]:
T0=$(date +%s.%N); curl -sb /tmp/cj -X DELETE -H 'X-Tunnex-CSRF: 1' \
  localhost/api/v1/organizations/$ORG/members/<bob-user-id>; echo "t0=$T0"
# t1 = A->B first drop; delta < 5s
# [GW]:
$NS nft list ruleset | grep 10.99.0.20     # -> EMPTY (Bob's /32 gone though his DEVICE was never revoked)
```

**Proof 7 — staleness + kernel rollback:** [GW]
```bash
$NS nft list ruleset > /tmp/R0            # baseline ruleset
docker compose exec postgres psql -U tunnex -d tunnex -tAc \
  "select capabilities->>'policy_hash' from nodes where status='active';"   # H0
# break the next apply (nft lives in the node container):
docker compose exec node-agent sh -c 'chmod -x $(command -v nft)'
# [CP] push any policy change (add a rule in the dashboard) -> new hash H1 pushed; note T0=now
# observe on CP (repeat this query):
docker compose exec postgres psql -U tunnex -d tunnex -c \
  "select capabilities->>'policy_hash' hash, capabilities->>'policy_error' err, capabilities->>'policy_hash_since' since from nodes where status='active';"
#   EXPECT NOW: err non-empty; hash == H0 (fail-closed); on [GW] `$NS nft list ruleset` == /tmp/R0 (rollback); A<->B still flows
# STALE timing (surface = the query above): compute now - since.
#   at T0+60s: now-since < 90s  -> NOT stale (no premature alarm)   [record]
#   at T0+95s: now-since > 90s  -> STALE                            [record t_stale]
# recover:
docker compose exec node-agent sh -c 'chmod +x $(command -v nft)'   # -> applies H1, err clears
```

**Proof 7b — flush→reconcile:** [GW]
```bash
docker compose exec node-agent nft flush chain ip tunnex forward
$NS nft list ruleset          # forward chain now only 'policy drop' -> A->B DROPS
# wait one reconcile interval -> allows reappear, A<->B flows again
```

**Proof 8 — mixed-mode both ways:** [A] ping B AND C in loops.
```bash
# off -> enforcing (admins->servers grant present):
T0=$(date +%s.%N); curl -sb /tmp/cj -X PUT -H 'content-type: application/json' -H 'X-Tunnex-CSRF: 1' \
  -d '{"mode":"enforcing"}' localhost/api/v1/organizations/$ORG/zero-trust-mode; echo "t0=$T0"
#   A<->B: ZERO lost pings across the flip (granted, atomic swap);  A->C: first drop, delta < 5s
# enforcing -> off:
T0=$(date +%s.%N); curl -sb /tmp/cj -X PUT -H 'content-type: application/json' -H 'X-Tunnex-CSRF: 1' \
  -d '{"mode":"off"}' localhost/api/v1/organizations/$ORG/zero-trust-mode; echo "t0=$T0"
#   A->C reachable again, delta < 5s
```
