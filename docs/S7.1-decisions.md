# S7.1 — Zero Trust policy MODEL (EPIC 7, story 1) — decisions (decision-first)

EPIC 7 = Zero Trust Access, **enterprise-gated**. Sequential build: **S7.1 model → S7.2 enforcement →
S7.3 device posture → S7.4 policy UI.** This story is the **data model + policy compiler ONLY**. No
data-plane change, no agent code, no UI — those are S7.2/S7.4. The compiler is the security-critical
core, so it is proven here in pure Go tests before anything enforces it on the wire.

## Grounding (what exists today — from a code sweep)

- **Enforcement point (for S7.2, not this story):** the gateway's nftables `forward` chain
  (`apps/node/internal/egress/egress_linux.go` `ruleset()`) is `policy drop` + a **blanket
  `iifname wg0 oifname wg0 accept`** (device↔device) + `wg0→!wg0 accept` (egress). Zero Trust =
  replace the blanket allows with **compiled per-source→per-destination allows, default-deny**.
- **Push vehicle (for S7.2):** `reconcile.DesiredState{Peers}` (`apps/node/internal/reconcile`),
  built control-plane-side by `nodes.Service.DesiredState()` from `ListActivePeersForNode` — each
  device peer gets exactly its `assigned_ip/32`. The compiled policy will ride this same channel.
- **WireGuard AllowedIPs is cryptokey routing, NOT a firewall** — so enforcement is nftables in the
  forward chain, not AllowedIPs manipulation. (AllowedIPs stays per-/32.)
- **Data model is GREENFIELD for policy:** NO groups (memberships + owner/admin/member roles only),
  NO resources/rules/ACL tables. Devices are `org_id`+`user_id`(owner, NOT NULL)+`node_id`(one
  gateway, NOT NULL)+`assigned_ip`(org-unique)+`status`. Users are global; membership binds user↔org.
- **Patterns to conform to:** enterprise build tag + `enterprise/<feature>` package; router port
  `nil` in open → `editionRequired()` 403 `edition_required` (SSO precedent, `sso_wire_{open,
  enterprise}.go`); RBAC `"resource:action"` perms + `make generate-rbac` drift guard; `writeAudit`
  in-tx, actor-from-ctx, secret-free metadata; migrations `NNNN_*.{up,down}.sql` (next = **0018**),
  `org_id NOT NULL` + `UNIQUE(org_id,…)`; OpenAPI-first + `make generate-check`.

## The model (LOCKED unless flagged OPEN below)

Classic allow-list Zero Trust: **subjects → destinations, default-deny.**

### Tables (migration `0018_zero_trust.up.sql`, all `org_id NOT NULL`, tenant-scoped uniqueness)

1. **`groups`** — `(id, org_id, name, description, timestamps)`; `UNIQUE(org_id, name)`. A named set of
   users. The subject of every rule.
2. **`group_members`** — `(group_id, user_id)`; `UNIQUE(group_id, user_id)`. User↔group. (A user's
   group set is resolved via their org membership — a group only contains users who are org members.)
3. **`resources`** — `(id, org_id, name, cidr, protocol, port_low, port_high, timestamps)`;
   `UNIQUE(org_id, name)`. A named **static** destination: a CIDR + optional L4 (protocol in
   `{any,tcp,udp}`, port range; `any`/null = all ports). Covers LAN subnets behind a gateway, egress
   ranges, specific hosts (`/32`). `0.0.0.0/0` = "the internet" (egress grant).
4. **`policy_rules`** — `(id, org_id, src_group_id, dst_kind, dst_id, timestamps)`;
   `dst_kind ∈ {resource, group}`. An **allow** grant: members of `src_group` may reach `dst`. When
   `dst_kind=group`, the destination resolves to that group's members' **devices' assigned_ips**
   (device-to-device); when `resource`, to the resource's cidr:ports. No `deny` rows — **default-deny
   is implicit** (see OPEN-B).
5. **Org flag `zero_trust_mode`** on `organizations` — `enum {off, enforcing}`, **default `off`**.
   `off` = today's blanket-allow mesh (zero behavior change on upgrade). `enforcing` = default-deny +
   compiled allows. Flipping it is an explicit, **audited** admin action (see OPEN-C).

### The compiler (`internal/enterprise/policy`, pure + exhaustively tested — the heart of S7.1)

`Compile(orgPolicy, devices, groupMembers) → map[nodeID] → []AllowEntry` where
`AllowEntry{SrcIP (device /32), DstCIDR, Protocol, PortLow, PortHigh}`.

- Resolves each device → owner user → user's groups → those groups' allow-rules → destinations
  (resource cidr:ports, or peer-group members' device /32s).
- Emits one allow entry per (device source ip, destination) pair. **Anything not emitted is denied**
  by S7.2's forward chain. Established/related return traffic is handled at the nftables layer (conntrack),
  not modeled here.
- **Identity binding preserved:** a device's access derives ONLY from its owning user's group
  memberships — no floating grants (the cross-cutting rule). A revoked/other-status device emits
  nothing.
- Pure function of DB state → deterministic, snapshot-testable. This is where default-deny is proven.

### API (enterprise-gated, `internal/enterprise/policy` + `http` port `nil` in open → 403)

CRUD under `/api/v1/organizations/{orgId}/`: `groups`, `groups/{id}/members`, `resources`, `policies`
(rules), and `PUT …/zero-trust-mode`. All mutations `authorize()` → `PermPolicyManage`; reads →
`PermPolicyView`. Open build: every endpoint returns `403 edition_required` (nil port). New RBAC
perms `policy:view` (owner/admin/member-read) + `policy:manage` (owner/admin), `make generate-rbac`
regenerates the drift fixture. Every mutation `writeAudit` (secret-free): `group.created`,
`group.member_added`, `resource.created`, `policy.rule.created/deleted`, `org.zero_trust_mode_changed`.

## LOCKED decisions (signed off) + their recorded conditions

**A — destination kinds = BOTH `resource` + `group`.** Device-to-device is a core case; compiler
resolves a `group` dst to its members' device `/32`s. **Conditions:**
- (A1) Group→/32 resolution makes compiled policy **device- and membership-dependent** → **device
  revoke, device create, AND membership change must all invalidate/recompile** (model-layer, so
  S7.1's — see D4). The **revocation full-sweep now also removes the revoked device's `/32` from every
  compiled ruleset where it appears as a DESTINATION**, within the <5s push spec — else S3.5 address
  reuse lets a recycled IP inherit the old device's grants.
- (A2) Deliberate-red: revoke a group-member device → prove its `/32` LEAVES the compiled output.

**B — allow-only rules, implicit default-deny.** No `effect` column. Deny/override **DEFERRED, not
rejected** — trigger = first real exception carve-out need (e.g. "all admins except contractor-admins");
if it lands it comes with its own decide-before-code on precedence semantics. **Conditions:**
- (B1) The default-deny deliberate-red proves the **BASE, not a rule**: compiler given **zero grants
  emits an EMPTY allow-set**, and the test asserts **empty ≠ permissive**.
- (B2) **Inverse guard:** the compiler must be **structurally incapable of emitting a blanket allow** —
  it can never reproduce the current `wg0→wg0` blanket-accept that S7.2 replaces. Assert it.

**C — org-level `zero_trust_mode {off,enforcing}`, default `off`.** Per-node/per-group granularity
DEFERRED (trigger = real mixed-fleet need; layerable later as a NARROWING of org-enforcing, not a
schema break). **Conditions:**
- (C1) Transitions **audited BOTH directions** — `org.zero_trust_enabled` / `org.zero_trust_disabled`,
  actor-attributed, same-tx.
- (C2) **Disabling re-opens the mesh (security-sensitive)** → gate it; **decided permission =
  `policy:manage` (owner/admin)** for the mode PUT, so the RBAC matrix + 401-walk arm with the endpoint.
- (C3) **Enable-with-zero-grants lockout guard = warn-and-confirm, server still obeys** (server is the
  truth, S4.7 precedent). NOT refuse-empty — locked-down-by-default is a legitimate posture. (The
  warn is a S7.4 UI concern; S7.1 records the server-obeys stance.)
- (C4) **Mode is a compiler INPUT, not enforcement special-casing:** `off` → compiler emits the legacy
  **blanket-mesh** ruleset; `enforcing` → compiler emits the **grant set**. ONE code path, ONE output
  artifact, mode just selects what's compiled. Both modes + the transition are pure-Go testable now
  (off→enforcing and enforcing→off both recompile; the latter is the fast lockout-recovery path).
- (C5) Deliberate-red: org `enforcing` + zero grants → compiled output has **no allow between device
  `/32`s** (pairs with B1). Flag lives on the `organizations` row → query-lint `org_id` guard covers
  its reads automatically; **confirm it is NOT added to the `globalTables` allowlist**.

**D — S7.1 = model + compiler ONLY; stops before push/enforce** (S7.2 wires DesiredState + agent
nftables + the on-the-wire proof). **Conditions:**
- (D1) **The compiled-policy artifact TYPE is an S7.1 deliverable, defined NOW** — the typed, versioned
  structure S7.2 drops into `DesiredState` after `ListActivePeersForNode()`. **Deterministic: same
  inputs → byte-identical output** (so reconcile stays a steady-state no-op per the reconcile-
  idempotence guard). Covers both modes (C4). S7.2 consumes it unchanged; any reshape is a RECORDED
  amendment, not a silent refactor.
- (D2) **Guards arm in S7.1:** new CRUD endpoints (groups/resources/rules/mode) join the **401-walk +
  RBAC matrix**; all new tables tenant-owned under the **query-lint `org_id` + `deleted_at` guards** (no
  `globalTables` additions); enterprise-gating per the S4.5 precedent (**403 `edition_required`** on open)
  via `policy_wire_{open,enterprise}.go`; migration `0018` tables with `updated_at` get the
  `set_updated_at` trigger (schema guard auto-catches). **One deliberate-red per new guard.**
- (D3) **DoD honesty — SUBSTITUTES ≠ SATISFIES:** S7.1's pure-Go compiler tests (default-deny empty-
  output red, group-member-revoke sweep red, determinism, both-modes) **substitute for but DO NOT
  satisfy** the on-the-wire proof. The wire proof (blanket `wg0→wg0` deleted, compiled default-deny
  enforced, a real packet DROPPED) is **S7.2's proof obligation — ledgered now** so it can't be quietly
  assumed proven.
- (D4) **Recompile-invalidation is model-layer → S7.1's:** device revoke/create + membership change →
  recompile-invalidation logic + tests land HERE; S7.2 only adds the push. The <5s revocation latency
  is measured end-to-end in S7.2, but the **invalidation hook must exist + be tested in S7.1**.

## DoD (S7.1)
- Migration `0018` (up+down, tenant-scoped, reversible, `updated_at` triggers) + sqlc queries + gen code.
- **Compiled-policy artifact type (D1)** — versioned, deterministic (same inputs → byte-identical),
  both-mode-capable; the exact struct S7.2 will push. Determinism test.
- `internal/enterprise/policy`: model + `Compile()` with exhaustive unit tests, incl. the **deliberate-
  reds**: (B1) zero grants → **empty** allow-set, empty≠permissive; (B2) compiler **cannot** emit a
  blanket allow; (A2) revoke a group-member device → its `/32` leaves the output (as src AND as dst);
  (C5) `enforcing` + zero grants → no device↔device allow; (C4) `off`→blanket-mesh vs `enforcing`→grants
  + both transition directions recompile; identity-binding (access only via owner's groups); port/proto
  scoping.
- **Recompile-invalidation hook (D4)** — device revoke/create + membership change invalidate/recompile;
  tested here (push is S7.2).
- Enterprise-gated CRUD API (OpenAPI-first) for groups / group-members / resources / rules + the
  **mode PUT gated by `policy:manage`** (C2); `policy:view` + `policy:manage` RBAC perms + drift
  fixtures regenerated.
- Audit on every mutation (secret-free, actor-attributed) incl. **both mode directions** (C1); test
  that metadata carries nothing sensitive.
- Open build: every policy endpoint 403 `edition_required` (edition test, SSO precedent). New endpoints
  join the **401-walk**; new tables under **query-lint `org_id`+`deleted_at`** with **no `globalTables`
  additions** (C5) — one deliberate-red per new guard (D2).
- `make generate-check` + both-edition build/test green. **No agent/data-plane change in this story**
  (the on-the-wire default-deny proof is S7.2's obligation, D3).
