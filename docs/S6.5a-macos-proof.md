# S6.5a macOS proof runbook (unsigned .dmg)

Runs the three S6.5a macOS proofs against the packaged artifact
`apps/client/release/Tunnex-0.1.0-universal.dmg`. Needs a reachable tenant + an active
gateway (the same VM/stack used for the S6.3/S6.4 smokes). GUI steps — run at the machine.

Rebuild the artifact any time with: `cd apps/client && npm run pack:mac`.

---

## Pre: a live tenant + gateway
The desktop app connects to a Tunnex server and needs an **active gateway** enrolled on
your org (so a device can be minted). Bring up the tenant + enroll a gateway as in the
S6.3 cloud/VM smoke. Note the server URL you'll type into the app.

---

## Proof A — Acceptance: install → connect → ping, ZERO terminal
The headline bar. Nothing about this should touch a terminal (except the optional hash check).

1. **Verify the download** (optional, from repo): `shasum -a 256 apps/client/release/Tunnex-0.1.0-universal.dmg` == the line in `apps/client/release/SHA256SUMS`.
2. **Install past Gatekeeper** (docs/install.md): open the `.dmg`, drag **Tunnex** to
   Applications. Launch: macOS 15 → **System Settings → Privacy & Security → Open Anyway**;
   macOS ≤14 → right-click → Open → Open.
3. **Sign in:** enter the server URL → sign in.
4. **Connect:** click **Connect**. A **native password prompt** appears once ("Tunnex needs
   to install its VPN helper") → enter your password.
   - **PASS:** the helper installs with no terminal; status goes **Connected** with an IP.
5. **Ping through the tunnel:** confirm connectivity to an org-network address (from the app's
   perspective / a browser to an internal host).
   - **PASS (headline):** installed → connected → reachable, **zero terminal commands**.

Diagnose (only if it fails): helper log at `/tmp/tunnex-helper.log` should show
`caller-auth: native`; `sudo launchctl print system/io.tunnex.helper` shows the daemon.

---

## Proof B — Packaged residue smoke (interim LaunchDaemon path)
After Proof A, prove uninstall leaves nothing.

1. Quit Tunnex. Drag **Tunnex.app** to the Trash.
2. Remove the helper: `sudo bash scripts/macos-uninstall.sh`.
   - **PASS:** the script's residue check prints OK for every line — no plist, daemon not
     loaded, `/usr/local/tunnex` gone, socket gone, `/etc/pf.conf` clean (anchor removed,
     restored from backup).

(SMAppService-based install/uninstall + its residue smoke is S6.5b — needs Developer ID.)

---

## Proof C — Tray / notification live display
With the app running (reinstall if you did Proof B):

1. **Tray icon** present in the menu bar; its menu + tooltip show the current state.
2. **Connect** → tray shows Connecting… then Connected; a **notification** fires.
3. **Disconnect** → tray → Not connected; notification fires.
4. Force a **fail-closed**: `sudo kill -9 $(pgrep -f tunnex-helper)` while connected →
   tray shows "Tunnel failed — kill-switch active" + a notification.
5. **Revoke** the device from the admin UI while connected → within ~30s the app auto-
   disconnects, tray shows "Device revoked", a notification fires.
6. Tray **click shows the window**; **Quit** exits.
   - **PASS:** tray + notifications track connect / disconnect / failed / revoked live,
     mirroring the window (connecting vs connected).

## Result — ALL THREE PASS (live, 2026-07-10, Apple Silicon)
- **Proof A PASS:** unsigned universal `.dmg` → install → sign in → Connect → one GUI admin
  prompt installed the helper → **Connected, IP 10.99.0.2, live handshake + rx/tx**, zero
  terminal. The in-app osascript privileged install worked end-to-end.
- **Proof B PASS:** `macos-uninstall.sh` residue check all-OK — no plist, daemon not loaded,
  `/usr/local/tunnex` gone, socket gone, `pf.conf` clean.
- **Proof C PASS:** tray menu = `Tunnex — Connected / Disconnect / Show / Quit`; content
  notifications fired on connect + disconnect; `kill -9` → "tunnel failed (kill-switch
  active)"; admin revoke → within ~30s the app auto-disconnected to "Device revoked —
  reconnect to re-enroll" (S6.4 revocation-teardown proven live end-to-end).
